Rushing the first deadline 101

September 6, 2019

Sometimes, you get the work done and everything's great. Those days are very rare for me.

This first lab required the installation , configuration, and usage of Docker to run some executables provided to us in the sandbox server. I've made docker containers before but I've been having multiple problems, not all of which will be given images.

Currently, I'm receiving multiple permissions issues with the commands given in the PDF. Specifically, starting from the class_start problem. On the sandbox server, I wanted to see if something was wrong with read/write permissions on the ""~/cit160/etc/profile" file. I chmod 777'd it, but "class_start" still does not work for me. Do not worry, I believe I changed the permissions back to before...

Fortunately speaking, I was able to get to the end, though i'm still reaching permissions issues... Funnily enough, this is not the first time i've been excited to talk about the common misconception with touch. 'Touch' as a command simply edits the access and modification times of a file. It does not necessarily create them, contrary to popular belief.
